Technical Deep Dive

Secure Authentication Logic

Proses login dan signup menggunakan enkripsi pada level backend untuk melindungi data sensitif user. Sistem memastikan setiap sesi valid sebelum mengizinkan modifikasi pada koleksi buku di database.

Node.js & Express Middleware

Arsitektur backend memanfaatkan **Middleware** untuk melakukan sanitasi input (body parsing). Setiap request `POST` untuk menambah buku melewati tahap verifikasi struktur data JSON sebelum akhirnya diterima oleh database.

MongoDB Compass Debugging

Pemanfaatan **MongoDB Compass** sebagai GUI mempermudah analisis performa index dan pengecekan redundansi data. Ini memastikan interaksi antara *Mongoose model* dan *Local Database* berjalan efisien tanpa *latency*.

RESTful API Lifecycle

Sistem ini mengelola *state* data secara asinkron. Saat buku ditambahkan, API mengirimkan status `201 Created` beserta objek buku terbaru, yang kemudian dirender secara instan di tabel tanpa perlu memuat ulang seluruh halaman.
